================================================================
  ダウンロード自動振り分けアプリ
================================================================

【動作要件】

  対応OS：
    macOS 12 Monterey 以降（動作確認済み・推奨）
    Windows 10 / 11（コード上は動作可能だが動作確認外）
    Linux（Ubuntu 20.04 以降など、tkinter が使える環境）

  ※ Windows の場合：
    - Python 公式インストーラ（python.org）を使うと tkinter が同梱される
    - brew は不要。watchdog は pip install watchdog でインストール可能
    - 監視フォルダのデフォルトは ~/Downloads（自動で展開される）
    - 多重起動防止の動作が macOS と若干異なる場合がある
    - 動作保証はしていないため、macOS での使用を推奨

  必須環境：
    Python        3.10 以上
    pip           最新推奨
    tkinter       Python に付属（macOS は別途 brew install が必要な場合あり）
    watchdog      2.0 以上（pip install watchdog でインストール）
    空きディスク  10MB 以上
    インターネット 不要（完全ローカル動作）


----------------------------------------------------------------
【インストール方法】

■ ワンクリックインストール（推奨）

1. install.command を右クリック →「開く」をクリック
   ※ 初回のみ Gatekeeper の警告が出る場合があります。「開く」で進んでください。

2. ターミナルが開き、自動で以下を実行します：
   - Python 3.10 以上の確認
   - tkinter の確認（不足時は Homebrew で自動インストール）
   - 仮想環境（.venv）の作成
   - watchdog のインストール

3. 「インストール完了」と表示されたら準備完了です。

■ 手動インストール（上記がうまくいかない場合）

  $ python3 -m venv .venv
  $ source .venv/bin/activate
  $ pip install watchdog


----------------------------------------------------------------
【使用方法】

■ 起動（ワンクリック）

  kiro.command をダブルクリック

■ 起動（ターミナルから）

  $ source .venv/bin/activate
  $ python main.py

  監視フォルダの監視を開始します。
  新しいファイルが追加されると自動でポップアップが表示されます。
  終了するには Ctrl+C を押してください。

■ 設定画面を開く

  $ python main.py --settings

  または、振り分けポップアップの「⚙ 設定」ボタンからも開けます。

  設定画面でできること：
  - プロジェクトの追加・編集・削除
  - フォルダ参照ダイアログでパスを選択
  - 監視フォルダの変更（デフォルト: ~/Downloads）

■ プロジェクトの登録

  方法1：設定画面から登録（推奨）

  $ python main.py --settings

  1.「プロジェクトを追加 / 編集」欄の「名前」に任意の名前を入力
  2.「参照…」ボタンでフォルダを選択（名前は自動補完される）
  3.「追加 / 更新」をクリックして保存
  4. 一覧に追加されたことを確認して「閉じる」

  方法2：projects.json を直接編集

  プロジェクトルートの projects.json をテキストエディタで開き、
  以下の形式で追記する：

  {
    "プロジェクト名": "/絶対パスでフォルダを指定",
    "ProjectA": "/Users/yourname/Documents/ProjectA",
    "ProjectB": "/Users/yourname/Documents/ProjectB"
  }

  ※ パスは絶対パスで記載すること
  ※ フォルダが存在しない場合は振り分け時にエラーになる


----------------------------------------------------------------
■ ファイルの振り分け操作

  ポップアップが表示されたら：
  1. 一覧からプロジェクトを選択
  2.「振り分ける」をクリック
     → プロジェクトフォルダ内に YYYY-MM-DD フォルダが作成され
        ファイルが移動されます
  3.「振り分けしない」をクリックすると Downloads にそのまま残ります

■ ログの確認

  move_log.csv に操作履歴が記録されます。
  項目：日時 / ファイル名 / 移動先 / プロジェクト名


----------------------------------------------------------------
【ファイル構成】

  install.command  インストーラ（ダブルクリックで実行）
  kiro.command     起動スクリプト（ダブルクリックで実行）
  main.py          起動スクリプト（ターミナル用）
  projects.json    プロジェクト設定
  settings.json    監視フォルダ設定（自動生成）
  move_log.csv     操作ログ（自動生成）
  kiro.lock        多重起動防止ファイル（自動生成・削除）
  setup.sh         セットアップスクリプト（ターミナル用）
  src/             ソースコード
  docs/spec.md     仕様書
